NexaboxDrive API 接口文档

内部版本 | Web 1.0 兼容版 | v1.4 更新文件夹支持

一、 全局鉴权说明

所有私有接口请求均需要通过 HTTP Header 传递 Token 进行身份验证。公共分享接口无需此 Header。

Header 键 格式与说明
Authorization Bearer <您的Token>

二、 核心流程详细说明

1. 大文件切片上传步骤 (Upload Chunking)

系统采用前端切片直传 OSS 的方案：

  1. 计算切片： 按固定大小（250MB）划分分块。
  2. 分块伪装： 伪装成 image/png 格式以绕过限制。
  3. 循环上传： 获取 OSS 签名并 POST 二进制数据。
  4. 元数据提交： 上传完成后提交分块 URL 数组及目标虚拟路径至 /api/files。

2. 分块合并与流式下载 (Download Merging)

下载时不进行物理合并，通过 Service Worker 按顺序拉取分块流式写入。说明见 sw.js。

3. 文件/文件夹分享流程 (Universal Sharing) Update

允许将文件或整个文件夹生成公共访问链接：

  1. 统一生成： 请求 /api/share，传入文件或文件夹的 fileId。
  2. 递归读取： 如果分享的是文件夹，公共接口会自动扫描该路径下所有层级的子文件。
  3. 流式批量下载： 分享页利用 File System Access API 结合 Service Worker，将递归获取到的文件流直接写入本地对应目录。

4. 虚拟文件夹管理 (Virtual Folders) New!

网盘不维护真实的物理目录结构，而是通过元数据中的 path 字段实现逻辑上的文件夹分类：

  • 文件夹也是一种特殊的元数据记录（包含 isFolder: true 标记）。
  • 根目录路径固定为 /。
  • 子目录路径格式示例：/Documents/ 或 /Photos/2026/。

三、 接口列表

1. 获取文件列表 (List Files)

  • 接口路径： /api/files
  • 请求方式： GET
  • 查询参数： path (可选，默认为 /)
返回示例： 
[
  {
    "id": "FILE_DIR_1714000000000",
    "filename": "我的文档",
    "isFolder": true,
    "path": "/",
    "size": 0
  },
  {
    "id": "FILE_1714000000001",
    "filename": "readme.txt",
    "size": 1024,
    "path": "/",
    "chunks": ["https://oss-url/part1"]
  }
]

2. 创建文件夹 (Create Folder) New!

  • 接口路径： /api/folder
  • 请求方式： POST
请求参数： 
{
  "name": "新建文件夹",
  "path": "/"  // 该文件夹所在的父级目录
}

3. 保存文件元数据 (Save Metadata)

  • 接口路径： /api/files
  • 请求方式： POST
请求参数： 
{
  "filename": "movie.mp4",
  "size": 524288000,
  "chunks": ["url1", "url2"],
  "path": "/Movies/" // 文件所属的虚拟路径
}

4. 创建分享链接 (Create Share)

  • 接口路径： /api/share
  • 请求方式： POST
请求参数： 
{
  "fileId": "FILE_1714000000000",
  "password": "abcd" 
}

1. 获取公共分享信息 (Get Public Share Info) Update

  • 接口路径： /api/public/share
  • 请求方式： GET / POST
  • 鉴权： 无需 Token

Step 1: GET 探测 (查询基础信息)
请求示例：GET /api/public/share?id=a1b2c3d4

{
  "filename": "我的照片墙",
  "size": 0,
  "isFolder": true,      // 标识是否为文件夹分享
  "needPassword": true   // 是否需要提取码
}

Step 2: POST 验证 (获取完整数据)
请求示例：POST /api/public/share 内容：{"id":"a1b2c3d4", "password":"..."}

如果是文件夹分享，返回示例： 
{
  "filename": "我的照片墙",
  "isFolder": true,
  "fullPath": "/photos/",  // 文件夹在网盘中的绝对路径
  "children": [            // 递归包含的所有子项
    {
      "filename": "IMG_001.jpg",
      "path": "/photos/2024/",
      "size": 5242880,
      "chunks": ["https://..."]
    },
    {
      "filename": "假期作业.pdf",
      "path": "/photos/",
      "size": 1048576,
      "chunks": ["https://..."]
    }
  ]
}

* 注意：单文件分享的返回格式保持原有 1.3 版本结构，不再赘述。

四、 客户端接入示例 (C#)

1. C# 提交元数据示例 (含路径支持)

// 在上传分片完成后，提交包含 path 的元数据
var metaData = new
{
    filename = originalName,
    size = totalSize,
    chunks = chunkUrls,
    path = "/Backup/2026/" // 指定存储路径
};
var jsonContent = new StringContent(JsonSerializer.Serialize(metaData), Encoding.UTF8, "application/json");
var metaRes = await client.PostAsync($"{API_BASE}/api/files", jsonContent);

2. C# 创建文件夹示例

public async Task CreateFolderAsync(string folderName, string parentPath)
{
    var content = new StringContent(JsonSerializer.Serialize(new { 
        name = folderName, 
        path = parentPath 
    }), Encoding.UTF8, "application/json");
    
    var response = await client.PostAsync($"{API_BASE}/api/folder", content);
    response.EnsureSuccessStatusCode();
}
