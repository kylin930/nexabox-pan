NexaboxDrive API接口文档

最后更新时间：2026年4月

一、 全局鉴权说明

所有接口请求均需要通过HTTPHeader传递Token进行身份验证。

Header键 格式与说明
Authorization Bearer <您的Token>

注意：统一账号系统会校验用户是否拥有all或NexaboxDrive权限。

二、 接口列表

1. 获取文件列表 (List Files)

  • 接口路径： /api/files
  • 请求方式： GET
  • 接口描述： 获取系统中已记录的所有文件信息。
返回示例： 
[
  {
    "id": "FILE_1714000000000",
    "filename": "example_video.mp4",
    "size": 524288000,
    "chunks": [
      "https://teachermate.oss-cn-qingdao.aliyuncs.com/chunk-001.png",
      "https://teachermate.oss-cn-qingdao.aliyuncs.com/chunk-002.png"
    ]
  }
]

2. 保存文件记录 (Save File Meta)

  • 接口路径： /api/files
  • 请求方式： POST
  • 接口描述： 在前端完成 OSS 上传后，提交文件元数据进行保存。
请求数据： 
{
  "filename": "my_file.zip",
  "size": 1024000,
  "chunks": [
    "https://oss-url-1...",
    "https://oss-url-2..."
  ]
}
返回示例： 
{
  "success": true,
  "fileId": "FILE_1714000000000"
}

3. 获取 OSS 上传签名 (Get OSS Signature)

  • 接口路径： /api/signature
  • 请求方式： GET
  • 接口描述： 从服务器获取直传阿里云 OSS 所需的临时签名和 Policy。
返回示例： 
{
  "accessKeyId": "LTAI5tXXXXXX",
  "signature": "base64_encoded_signature_string",
  "policy": "base64_encoded_policy_json",
  "host": "https://teachermate.oss-cn-qingdao.aliyuncs.com/"
}
